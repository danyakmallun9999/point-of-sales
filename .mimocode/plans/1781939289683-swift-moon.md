# Plan: Improve Test Coverage for Brew & Bytes POS

## Goal
Increase test coverage from ~43 to ~139 tests by adding tests for all untested controllers, services, and model logic.

## Phase 2: Unit Tests (3 files, ~18 tests)

### `tests/Unit/Services/AprioriServiceTest.php` — 6 tests
- Returns empty array when no paid orders
- Generates recommendations from paid orders
- Each recommendation has required fields (antecedent, consequent, support, confidence)
- Respects minSupport threshold
- Respects minConfidence threshold
- Returns at most 5 recommendations

### `tests/Unit/Models/ProductFIFOTest.php` — 7 tests
- Deducts from single batch
- Deducts across multiple batches in FIFO order
- Fully exhausts a batch before moving to next
- Deducts stock from product aggregate
- Handles partial batch deduction
- Deducts exact batch quantity
- Deducts from three batches sequentially

### `tests/Unit/Models/OrderMarkAsPaidTest.php` — 5 tests
- Marks pending order as paid
- Deducts stock via FIFO on markAsPaid
- Does nothing if already paid (idempotent)
- Sets payment_status to paid
- Deducts stock for multiple order items

## Phase 3: Feature Tests — Management CRUD (3 files, ~40 tests)

### `tests/Feature/Management/CategoryTest.php` — 12 tests
- Admin/manager can view index (with product_count)

- Admin can create category (validates unique name, required fields)
- Admin can update category
- Admin can delete category without products
- Cannot delete category with products

### `tests/Feature/Management/ProductTest.php` — 14 tests
- Admin/manager can view index
- Cashier/guest cannot access
- Admin can create product (validates required fields, category exists)
- Creating product with stock creates InventoryBatch
- Admin can update product
- Admin can delete product without orders
- Cannot delete product that has been ordered

### `tests/Feature/Management/UserTest.php` — 14 tests
- Admin can view/create/update/delete users
- Manager/cashier cannot access (admin-only)
- Validates unique email, required fields, password confirmation, valid role
- Admin cannot delete themselves
